# Adding Projects

This guide explains how to add new project documentation to this hub.

## Overview

Each project in this documentation hub is configured as a separate Docusaurus docs plugin instance. This allows each project to have:
- Its own documentation structure
- Independent versioning
- Separate sidebar configuration
- Isolated content management

## Project Structure

All projects are located in the `projects/` directory:

```
projects/
  ├── fortihub/
  └── fortihub/
      ├── version-1.0.0/
      ├── versioned_sidebars/
      └── versions.json
```

## Step-by-Step Guide

### 1. Create Project Directory

Create a new directory for your project:

```bash
mkdir -p projects/your-project-name/docs
```

### 2. Create Sidebar Configuration

Create `projects/your-project-name/sidebars.ts`:

```typescript
import type { SidebarsConfig } from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  yourProjectSidebar: [{ type: 'autogenerated', dirName: '.' }],
};

export default sidebars;
```

### 3. Add Initial Documentation

Create an intro page at `projects/your-project-name/docs/intro.md`:

```markdown
---
sidebar_position: 1
---

# Introduction

Welcome to Your Project Name documentation!

## Overview

Brief description of your project...
```

### 4. Update Docusaurus Configuration

In `docusaurus.config.ts`, add a new plugin instance:

```typescript
plugins: [
  // ... existing plugins
  [
    '@docusaurus/plugin-content-docs',
    {
      id: 'your-project-name',
      path: 'projects/your-project-name/docs',
      routeBasePath: 'your-project-name',
      sidebarPath: './projects/your-project-name/sidebars.ts',
      versions: {
        current: {
          label: 'Next',
          path: 'next',
        },
      },
      includeCurrentVersion: true,
    },
  ],
],
```

### 5. Add Navbar Item

Add your project to the navbar dropdown in `docusaurus.config.ts`:

```typescript
{
  type: 'dropdown',
  label: 'Projects',
  position: 'left',
  items: [
    {
      to: '/fortihub/intro',
      label: 'FortiHub',
    },
    {
      to: '/your-project-name/intro',
      label: 'Your Project Name',
    },
  ],
},
```

### 6. Update Homepage

Add a project card in `src/pages/index.tsx`:

```tsx
<ProjectCard
  title="Your Project Name"
  description="Brief description of your project"
  to="/your-project-name/intro"
/>
```

### 7. Test Your Changes

Start the development server:

```bash
npm start
```

Navigate to `http://localhost:3000/your-project-name/intro` to verify your documentation appears correctly.

## Best Practices

- **Use descriptive titles**: Make it easy for users to understand what each page covers
- **Organize logically**: Group related content together
- **Add examples**: Include code examples and use cases
- **Keep it updated**: Regularly update documentation as your project evolves
- **Use versioning**: Create versions for major releases (see [Versioning Guide](../versioning.md))
