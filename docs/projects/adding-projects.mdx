# Adding Projects

<ProjectProvider>

This guide explains how to add new project documentation to this nexus. 

<ProjectInputs />

Each project in this nexus is configured as a separate Docusaurus docs plugin instance. This allows each project to have:
- Its own documentation structure
- Independent versioning (Not configured at the moment)
- Separate sidebar configuration
- Isolated content management

## Project Structure

All projects are located in the `projects/` directory:

<DynamicCode language="text">
{({ projectId }) => `projects/
  ├── fortihub/
  └── ${projectId || '<proj-name>'}/
      ├── docs/
      ├── sidebars.ts
      └── versions.json`}
</DynamicCode>

## Step-by-Step Guide

### 1. Create Project Directory

Create a new directory for your project:

<DynamicCode language="bash">
{({ projectId }) => `mkdir -p projects/${projectId || '<proj-name>'}/docs`}
</DynamicCode>

### 2. Create Sidebar Configuration

Create `projects/<proj-name>/sidebars.ts`:

<DynamicCode language="bash">
{({ projectId }) => `touch projects/${projectId || '<proj-name>'}/sidebars.ts`}
</DynamicCode>

and add the following content:

<DynamicCode language="typescript">
{({ projectId, sidebarType }) => {
    const id = projectId || '<proj-name>';
    const camelId = id.replace(/-./g, x=>x[1].toUpperCase()) + 'Sidebar';
    
    if (sidebarType === 'manual') {
        return `import type { SidebarsConfig } from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  ${camelId}: [
    'intro',
    // Add your document IDs here
  ],
};

export default sidebars;`;
    }

    return `import type { SidebarsConfig } from '@docusaurus/plugin-content-docs';

const sidebars: SidebarsConfig = {
  ${camelId}: [{ type: 'autogenerated', dirName: '.' }],
};

export default sidebars;`;
}}
</DynamicCode>

### 3. Add Initial Documentation

Create an intro page at `projects/<proj-name>/docs/intro.md`:

<DynamicCode language="markdown">
{({ projectName }) => `---
sidebar_position: 1
---

# Introduction

Welcome to ${projectName || 'Your Project Name'} documentation!

## Overview

Brief description of your project...`}
</DynamicCode>

### 4. Update Docusaurus Configuration

In `docusaurus.config.ts`, add a new plugin instance. In the `plugins` array, add a new object:

<DynamicCode language="typescript">
{({ projectId }) => {
    const id = projectId || '<proj-name>';
    return `[
    '@docusaurus/plugin-content-docs',
    {
      id: '${id}',
      path: 'projects/${id}/docs',
      routeBasePath: '${id}',
      sidebarPath: './projects/${id}/sidebars.ts',
      editUrl: 'https://github.com/surajmgr/nexus/tree/main/',
    },
  ],`;
}}
</DynamicCode>

### 5. Add Navbar Item

Add your project to the navbar dropdown in `docusaurus.config.ts`. In the `themeConfig.navbar.items` array, find the `dropdown` item with label `Projects` and add a new object to the `items` array:

<DynamicCode language="typescript">
{({ projectId, projectName }) => `{
      to: '/${projectId || '<proj-name>'}/intro',
      label: '${projectName || 'Your Project Name'}',
},`}
</DynamicCode>

### 6. Update Projects page

Add a list item in `docs/projects/index.md`:

<DynamicCode language="markdown">
{({ projectId, projectName }) => `- [${projectName || 'Your Project Name'}](/${projectId || '<proj-name>'}/intro)`}
</DynamicCode>

### 7. Test Your Changes

Start the development server:

```bash
npm start
```

Navigate to `http://localhost:3000/<proj-name>/intro` to verify your documentation appears correctly.
